## PostHog: The Open-Source Platform for Product Analytics and More

**PostHog empowers you to build successful products with a comprehensive, open-source platform, offering essential tools for user behavior analysis, experimentation, and product development.**  [Explore the PostHog Repository](https://github.com/PostHog/posthog)

**Key Features:**

*   **Product Analytics:** Understand user behavior with event-based analytics, autocapture, and custom SQL queries.
*   **Web Analytics:** Monitor website traffic, user sessions, and key metrics like conversions and web vitals.
*   **Session Replays:** Watch real user sessions to diagnose issues and gain insights into user interaction.
*   **Feature Flags:**  Safely roll out new features with targeted releases.
*   **Experiments:**  Test and measure the impact of changes on your product, including no-code experimentation.
*   **Error Tracking:** Track errors, receive alerts, and resolve issues efficiently.
*   **Surveys:** Gather user feedback with no-code templates or custom survey builders.
*   **Data Warehouse:** Sync data from external tools to query alongside your product data.
*   **Data Pipelines:** Transform and route your data to over 25+ tools and any webhook.
*   **LLM Observability:** Monitor and analyze LLM-powered applications.

PostHog offers a generous free tier, and for more information on paid plans, please visit [PostHog's pricing page](https://posthog.com/pricing).

## Getting Started with PostHog Cloud (Recommended)

The fastest way to get started is by signing up for free on [PostHog Cloud](https://us.posthog.com/signup) or [PostHog Cloud EU](https://eu.posthog.com/signup). Enjoy a free tier with 1 million events, 5,000 recordings, 1 million flag requests, 100,000 exceptions, and 250 survey responses per month.

## Self-Hosting the Open-Source Hobby Deploy (Advanced)

Deploy a hobby instance with Docker on Linux in a single line:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/posthog/posthog/HEAD/bin/deploy-hobby)"
```

For open-source deployments, we recommend migration to [PostHog Cloud](https://posthog.com/docs/migrate/migrate-to-cloud) after ~100k events per month. See the [self-hosting docs](https://posthog.com/docs/self-host) for details.

## Setting Up PostHog

Install PostHog using the [JavaScript web snippet](https://posthog.com/docs/getting-started/install?tab=snippet), one of the [SDKs](https://posthog.com/docs/getting-started/install?tab=sdks), or through the [API](https://posthog.com/docs/getting-started/install?tab=api).

Available SDKs:

| Frontend                                              | Mobile                                                          | Backend                                             |
| ----------------------------------------------------- | --------------------------------------------------------------- | --------------------------------------------------- |
| [JavaScript](https://posthog.com/docs/libraries/js)   | [React Native](https://posthog.com/docs/libraries/react-native) | [Python](https://posthog.com/docs/libraries/python) |
| [Next.js](https://posthog.com/docs/libraries/next-js) | [Android](https://posthog.com/docs/libraries/android)           | [Node](https://posthog.com/docs/libraries/node)     |

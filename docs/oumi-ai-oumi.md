## Oumi: Your All-in-One Platform for Building and Deploying Cutting-Edge Foundation Models

Oumi is a fully open-source platform, providing everything you need to build, train, evaluate, and deploy state-of-the-art foundation models. **[Visit the Oumi GitHub repository](https://github.com/oumi-ai/oumi) for more information.**

### Key Features

*   **Simplified Model Lifecycle:** Manage every stage from data preparation and training to evaluation and deployment with a unified workflow.
*   **Diverse Model Support:** Works with a wide array of text and multimodal models, including Llama, DeepSeek, Qwen, and Phi families.
*   **Flexible Training Options:** Support for SFT, LoRA, QLoRA, DPO, and other cutting-edge techniques for models ranging from 10M to 405B parameters.
*   **Data Curation Tools:** Utilize LLM judges to synthesize and curate your training data effectively.
*   **Efficient Deployment:** Deploy models seamlessly using popular inference engines like vLLM and SGLang.
*   **Comprehensive Evaluation:** Evaluate models using standard benchmarks and Oumi's robust evaluation framework.
*   **Cloud-Ready:** Run Oumi on various platforms, from laptops to clusters and cloud providers such as AWS, Azure, GCP, and Lambda.
*   **API Integration:** Integrate open models and commercial APIs, including OpenAI, Anthropic, and Vertex AI.

### Installation

Install Oumi with these simple steps:

```bash
# Install the package (CPU & NPU only)
pip install oumi  # For local development & testing

# OR, with GPU support (Requires Nvidia or AMD GPU)
pip install oumi[gpu]  # For GPU training

# To get the latest version, install from the source
pip install git+https://github.com/oumi-ai/oumi.git
```

For detailed installation instructions, refer to the [installation guide](https://oumi.ai/docs/en/latest/get_started/installation.html).

### Oumi CLI

Use the `oumi` command-line interface for training, evaluation, and inference:

```bash
# Training
oumi train -c configs/recipes/smollm/sft/135m/quickstart_train.yaml

# Evaluation
oumi evaluate -c configs/recipes/smollm/evaluation/135m/quickstart_eval.yaml

# Inference
oumi infer -c configs/recipes/smollm/inference/135m_infer.yaml --interactive
```

For in-depth usage, see the [training](https://oumi.ai/docs/en/latest/user_guides/train/train.html), [evaluation](https://oumi.ai/docs/en/latest/user_guides/evaluate/evaluate.html), [inference](https://oumi.ai/docs/en/latest/user_guides/infer/infer.html), and [llm-as-a-judge](https://oumi.ai/docs/en/latest/user_guides/judge/judge.html) guides.

### Running Jobs Remotely

Launch and manage jobs remotely on cloud platforms:

```bash
# GCP
oumi launch up -c configs/recipes/smollm/sft/135m/quickstart_gcp_job.yaml

# AWS
oumi launch up -c configs/recipes/smollm/sft/135m/quickstart_gcp_job.yaml --resources.cloud aws

# Azure
oumi launch up -c configs/recipes/smollm/sft/135m/quickstart_gcp_job.yaml --resources.cloud azure

# Lambda
oumi launch up -c configs/recipes/smollm/sft/135m/quickstart_gcp_job.yaml --resources.cloud lambda
```

**Note:** Oumi is currently in beta, with ongoing development and feature improvements.

*   **Research-Ready:** Ideal for ML research with reproducible experiments.
*   **Wide Model Support:** Compatible with a broad range of models.
*   **State-of-the-Art Performance:** Support for distributed training and optimized inference.
*   **Community-Focused:** 100% open source with an active community.

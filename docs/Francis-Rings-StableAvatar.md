# StableAvatar: Generate Infinite-Length Audio-Driven Avatar Videos

StableAvatar offers a groundbreaking approach to create compelling, high-fidelity, and identity-preserving avatar videos driven by audio, empowering users to generate infinite-length videos without compromising quality. Check out the original repo for more details: [https://github.com/Francis-Rings/StableAvatar](https://github.com/Francis-Rings/StableAvatar)

## Key Features:

*   **Infinite-Length Video Generation:** Create videos of virtually unlimited duration.
*   **Identity Preservation:** Maintains the subject's identity throughout the video.
*   **High Fidelity:** Produces high-quality video output.
*   **No Post-Processing Required:** Generates videos directly without face-swapping or restoration tools.
*   **Audio-Driven Animation:** Syncs avatar movements seamlessly with audio input.
*   **Flexible Resolutions:** Supports 512x512, 480x832, and 832x480 resolutions.
*   **Efficient Inference:** Runs efficiently on modern GPUs, with options for reduced VRAM usage.
*   **Training Codes:** Detailed documentation to facilitate custom training, finetuning and Lora-training.

### ðŸ§± Base Model Inference

*   Run inference using the `inference.sh` script.  Customize the parameters in the script.

```bash
bash inference.sh
```

*   **Key Parameters:**
    *   `--width` and `--height`: Set the output resolution.
    *   `--output_dir`:  Specify the output directory.
    *   `--validation_reference_path`, `--validation_driven_audio_path`, `--validation_prompts`: Set paths for the reference image, audio, and text prompts.
    *   `--pretrained_model_name_or_path`, `--pretrained_wav2vec_path`, `--transformer_path`: Specify the paths to pre-trained models.
    *   `--sample_steps`, `--overlap_window_length`, `--clip_sample_n_frames`: Control inference steps, overlapping context, and frame count.
    *   `--sample_text_guide_scale` and `--sample_audio_guide_scale`:  Control prompt and audio guidance scales.

*   **Prompts:**  Use prompts in the format: `[Description of first frame]-[Description of human behavior]-[Description of background (optional)]`.

*   **Gradio Interface:** Run a Gradio interface with `python app.py`.

*   **Example Cases:**  Explore the provided examples in `path/StableAvatar/examples`.

#### ðŸ’¡Tips

*   Use the correct `--transformer_path` to switch between the two versions of Wan2.1-1.3B based weights.
*   Use `--GPU_memory_mode` to optimize GPU memory usage.
*   Use Multi-GPU inference options to speed up inference.
*   Convert the video without audio to MP4 with audio using `ffmpeg`.

*   StableAvatar is trained on mixed-resolution videos.
*   Use `ffmpeg` and refer to the provided code to extract frames and audio from the source data.
*   Refer to [StableAnimator repo](https://github.com/Francis-Rings/StableAnimator) to extract human face masks.
*   Use `lip_mask_extractor.py` to extract lip masks.
*   Run one of the training scripts:

```bash
# Training StableAvatar on a single resolution setting (512x512) in a single machine
bash train_1B_square.sh
# Training StableAvatar on a single resolution setting (512x512) in multiple machines
bash train_1B_square_64.sh
# Training StableAvatar on a mixed resolution setting (480x832 and 832x480) in a single machine
bash train_1B_rec_vec.sh
# Training StableAvatar on a mixed resolution setting (480x832 and 832x480) in multiple machines
bash train_1B_rec_vec_64.sh
```

*   For the parameter details of `train_1B_square.sh` and `train_1B_rec_vec.sh`, `CUDA_VISIBLE_DEVICES` refers to gpu devices.
*   `--pretrained_model_name_or_path`, `--pretrained_wav2vec_path`, and `--output_dir` refer to the pretrained Wan2.1-1.3B path, pretrained Wav2Vec2.0 path, and the checkpoint saved path of the trained StableAvatar.
*   `--train_data_square_dir`, `--train_data_rec_dir`, and `--train_data_vec_dir` are the paths of `video_square_path.txt`, `video_rec_path.txt`, and `video_vec_path.txt`, respectively.
*   `--validation_reference_path` and `--validation_driven_audio_path` are paths of the validation reference image and the validation driven audio.
*   `--video_sample_n_frames` is the number of frames that StableAvatar processes in a single batch.
*   `--num_train_epochs` is the training epoch number. It is worth noting that the default number of training epochs is set to infinite. You can manually terminate the training process once you observe that your StableAvatar has reached its peak performance.
*   For the parameter details of `train_1B_square_64.sh` and `train_1B_rec_vec_64.sh`, we set the GPU configuration in `path/StableAvatar/accelerate_config/accelerate_config_machine_1B_multiple.yaml`.

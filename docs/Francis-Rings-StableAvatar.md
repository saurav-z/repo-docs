## Key Features

*   **Infinite-Length Video Generation:** Synthesize videos of any length without degradation.
*   **Identity Preservation:** Maintain consistent avatar identity throughout the entire video.
*   **End-to-End Synthesis:** Generate videos directly, *without* the need for post-processing tools like face-swapping or restoration.
*   **High-Fidelity Results:** Produce high-quality, natural-looking avatar animations.
*   **Audio-Driven Animation:** Animate avatars using audio input for realistic lip-sync and expressive movements.
*   **Easy to Use:**  Quickstart guides and pre-built demos make it simple to get started.

## Overview

StableAvatar is a groundbreaking video diffusion transformer that addresses the limitations of existing audio-driven avatar video generation models.  It overcomes the challenges of synthesizing long videos with natural audio synchronization and identity consistency by introducing a novel Time-step-aware Audio Adapter, an Audio Native Guidance Mechanism, and a Dynamic Weighted Sliding-window Strategy.  The result is high-quality, infinite-length avatar videos directly synthesized from audio input.

### ğŸ§± Base Model Inference

Modify the `inference.sh` script:

```bash
bash inference.sh
```

Key parameters to adjust include:

*   `--width`, `--height`:  Output resolution (512x512, 480x832, or 832x480).
*   `--output_dir`: Save path.
*   `--validation_reference_path`, `--validation_driven_audio_path`, `--validation_prompts`: Paths for the reference image, audio, and prompts.
*   `--pretrained_model_name_or_path`, `--pretrained_wav2vec_path`, `--transformer_path`: Paths to the pre-trained models.
*   `--sample_steps`, `--overlap_window_length`, `--clip_sample_n_frames`: Inference parameters.
*   `--sample_text_guide_scale`, `--sample_audio_guide_scale`: CFG scales.

Run Gradio Interface:

```bash
python app.py
```

Example videos are available in `path/StableAvatar/examples`.

#### ğŸ’¡Tips

*   Use `--transformer_path` to switch between `transformer3d-square.pt` and `transformer3d-rec-vec.pt` model versions.
*   Use `--GPU_memory_mode` to manage GPU memory usage (`model_full_load`, `sequential_cpu_offload`, `model_cpu_offload_and_qfloat8`, `model_cpu_offload`).
*   Use `--ulysses_degree` and `--ring_degree` to speed up inference on multiple GPUs.

To generate a video with audio, use `ffmpeg`:

```bash
ffmpeg -i video_without_audio.mp4 -i /path/audio.wav -c:v copy -c:a aac -shortest /path/output_with_audio.mp4
```

Training Commands:

```bash
# Training StableAvatar on a single resolution setting (512x512) in a single machine
bash train_1B_square.sh
# Training StableAvatar on a single resolution setting (512x512) in multiple machines
bash train_1B_square_64.sh
# Training StableAvatar on a mixed resolution setting (480x832 and 832x480) in a single machine
bash train_1B_rec_vec.sh
# Training StableAvatar on a mixed resolution setting (480x832 and 832x480) in multiple machines
bash train_1B_rec_vec_64.sh
```

Key parameters in `train_1B_square.sh` and `train_1B_rec_vec.sh`:

*   `CUDA_VISIBLE_DEVICES`: GPU devices.
*   `--pretrained_model_name_or_path`, `--pretrained_wav2vec_path`, `--output_dir`: Pre-trained model paths and checkpoint save path.
*   `--train_data_square_dir`, `--train_data_rec_dir`, `--train_data_vec_dir`: Paths to data lists.
*   `--validation_reference_path`, `--validation_driven_audio_path`: Validation data paths.
*   `--video_sample_n_frames`: Frames per batch.
*   `--num_train_epochs`: Training epochs.

For multi-GPU training, modify the configuration files in `path/StableAvatar/accelerate_config/`.

**Important:**  Training requires around 50GB of VRAM.

Regarding training Wan2.1-14B-based StableAvatar, you can run the following command:

```bash
# Training StableAvatar on a mixed resolution setting (480x832, 832x480, and 512x512) in multiple machines

huggingface-cli download Wan-AI/Wan2.1-I2V-14B-720P --local-dir ./checkpoints/Wan2.1-I2V-14B-720P # Optional

```
You can also perform LoRA training:
```bash
# Training StableAvatar-1.3B on a mixed resolution setting (480x832 and 832x480) in a single machine

# Lora-Training StableAvatar-14B on a mixed resolution setting (480x832, 832x480, and 512x512) in multiple machines
bash train_14B_lora.sh
```
You can modify `--rank` and `--network_alpha` to control the quality of your lora training/finetuning.

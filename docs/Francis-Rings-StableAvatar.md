# StableAvatar: Generate Infinite-Length Avatar Videos from Audio

**StableAvatar revolutionizes audio-driven avatar video generation, enabling the creation of infinite-length, high-quality videos with remarkable identity preservation.**  See the original repo: [https://github.com/Francis-Rings/StableAvatar](https://github.com/Francis-Rings/StableAvatar)

**Key Features:**

*   **Infinite-Length Video Generation:** Generate videos of any length, limited only by your compute resources.
*   **ID-Preserving:**  Maintain the identity of the avatar throughout the video.
*   **High-Fidelity Results:** Produce videos with excellent visual quality and audio synchronization.
*   **End-to-End Solution:**  No reliance on post-processing tools like face-swapping or restoration.
*   **Time-step-aware Audio Adapter:** Prevents error accumulation in long video generation.
*   **Audio Native Guidance Mechanism:** Enhances audio synchronization.
*   **Dynamic Weighted Sliding-window Strategy:** Smooths the infinite-length videos.

## Overview

<p align="center">
  <img src="assets/figures/framework.jpg" alt="model architecture" width="1280"/>
  </br>
  <i>The overview of the framework of StableAvatar.</i>
</p>

StableAvatar overcomes the limitations of existing models, which struggle to generate long videos with consistent audio synchronization and identity.  It introduces a novel architecture that synthesizes high-quality, infinite-length videos without post-processing. Key innovations include a Time-step-aware Audio Adapter and an Audio Native Guidance Mechanism, along with a Dynamic Weighted Sliding-window Strategy.

## Quickstart Guide

This guide provides instructions for using the basic version of the StableAvatar model.

### üß± Environment Setup

```bash
pip install torch==2.6.0 torchvision==0.21.0 torchaudio==2.1.1 --index-url https://download.pytorch.org/whl/cu124
pip install -r requirements.txt
# Optional to install flash_attn to accelerate attention computation
pip install flash_attn
```

### üß± Environment Setup for Blackwell series chips

```bash
pip install torch==2.7.0 torchvision==0.22.0 torchaudio==2.7.0 --index-url https://download.pytorch.org/whl/cu128
pip install -r requirements.txt
# Optional to install flash_attn to accelerate attention computation
pip install flash_attn
```

### üß± Download Weights

Download the necessary weights using `huggingface-cli`. If you encounter issues, set the environment variable: `export HF_ENDPOINT=https://hf-mirror.com`.

```bash
pip install "huggingface_hub[cli]"
cd StableAvatar
mkdir checkpoints
huggingface-cli download FrancisRing/StableAvatar --local-dir ./checkpoints
```

Organize the weights as described in the original README.

### üß± Audio Extraction

Extract audio from video files:

```bash
python audio_extractor.py --video_path="path/test/video.mp4" --saved_audio_path="path/test/audio.wav"
```

### üß± Vocal Separation (Optional)

Separate vocals from audio for improved lip-sync:

```bash
pip install audio-separator[gpu]
python vocal_seperator.py --audio_separator_model_file="path/StableAvatar/checkpoints/Kim_Vocal_2.onnx" --audio_file_path="path/test/audio.wav" --saved_vocal_path="path/test/vocal.wav"
```

### üß± Base Model Inference

Run inference using the provided `inference.sh` script, adjusting parameters as needed.  Prompts are critical; use the format: `[Description of first frame]-[Description of human behavior]-[Description of background (optional)]`.

```bash
bash inference.sh
```

Key parameters in `inference.sh`:

*   `--width` and `--height`: Set video resolution (512x512, 480x832, or 832x480).
*   `--output_dir`:  Output directory for generated videos.
*   `--validation_reference_path`, `--validation_driven_audio_path`, `--validation_prompts`: Paths to reference image, audio, and text prompts.
*   `--pretrained_model_name_or_path`, `--pretrained_wav2vec_path`, `--transformer_path`: Paths to model weights.
*   `--sample_steps`, `--overlap_window_length`, `--clip_sample_n_frames`: Inference parameters.
*   `--sample_text_guide_scale` and `--sample_audio_guide_scale`:  CFG scales.

For a Gradio interface:

```bash
python app.py
```

Examples are available in the `examples` directory.

#### üí°Tips

*   Use the correct `--transformer_path` for the desired resolution (square or vec/rec).
*   Manage GPU memory with `--GPU_memory_mode` (e.g., `model_cpu_offload` for lower VRAM usage).
*   Use Multi-GPU inference with `--ulysses_degree` and `--ring_degree`.
*   Convert output to MP4 with audio using `ffmpeg`:

```bash
ffmpeg -i video_without_audio.mp4 -i /path/audio.wav -c:v copy -c:a aac -shortest /path/output_with_audio.mp4
```

### üß± Model Training

Follow the data organization and training instructions in the original README for training your own StableAvatar model or finetuning the model.

*Training requires specific data formatting and significant VRAM.*

### üß± Model Finetuning
See the original README for commands for finetuning.

### üß± VRAM Requirement and Runtime

For a 5-second video (480x832, fps=25), the basic model requires approximately 18GB VRAM and takes about 3 minutes on a 4090 GPU.

**Note**: The 3D VAE decoder can significantly increase memory usage for long videos.

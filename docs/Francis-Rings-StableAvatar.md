StableAvatar is a groundbreaking end-to-end video diffusion transformer that generates high-quality, identity-preserving avatar videos from audio input, without the need for post-processing.

**Key Features:**

*   **Infinite-Length Video Generation:** Synthesize avatar videos of any length.
*   **Audio-Driven:** Directly driven by audio input for natural lip-sync and expression.
*   **Identity Preservation:** Maintains the subject's identity throughout the video.
*   **End-to-End Solution:** Generates videos directly, eliminating the need for face-swapping or restoration tools.
*   **High Fidelity:** Produces high-quality, visually appealing videos.

## Overview

<p align="center">
  <img src="assets/figures/framework.jpg" alt="model architecture" width="1280"/>
  </br>
  <i>The overview of the framework of StableAvatar.</i>
</p>

StableAvatar overcomes the limitations of existing audio-driven avatar video generation models, providing a solution for synthesizing long, high-quality videos with accurate audio synchronization and identity consistency.  The core innovation lies in a novel Time-step-aware Audio Adapter to prevent error accumulation and an Audio Native Guidance Mechanism to enhance audio synchronization.


Modify the parameters in `inference.sh` to customize your video generation:

*   `--width` and `--height`:  Set the video resolution (512x512, 480x832, or 832x480).
*   `--output_dir`:  Specify the output directory for the generated video.
*   `--validation_reference_path`, `--validation_driven_audio_path`, `--validation_prompts`:  Set the paths for the reference image, audio, and text prompts.  Prompts should follow the format:  `[Description of first frame]-[Description of human behavior]-[Description of background (optional)]`.
*   `--pretrained_model_name_or_path`, `--pretrained_wav2vec_path`, `--transformer_path`:  Set the paths for the pretrained model weights.
*   `--sample_steps`, `--overlap_window_length`, `--clip_sample_n_frames`:  Adjust these parameters to optimize video quality and generation speed.
*   `--sample_text_guide_scale`, `--sample_audio_guide_scale`:  Control the guidance scale.

#### ğŸ’¡ Tips

*   Experiment with different `--transformer_path` settings (`transformer3d-square.pt` or `transformer3d-rec-vec.pt`) for different resolution support.
*   Use `--GPU_memory_mode` to manage GPU memory usage.
*   Utilize Multi-GPU inference for faster processing.
*   Use `ffmpeg` to add audio to the generated video.

```bash
ffmpeg -i video_without_audio.mp4 -i /path/audio.wav -c:v copy -c:a aac -shortest /path/output_with_audio.mp4
```

*   **`video_square_path.txt`, `video_rec_path.txt`, `video_vec_path.txt`:** Text files listing the paths to the video clip folders.

```bash
huggingface-cli download Wan-AI/Wan2.1-I2V-14B-480P --local-dir ./checkpoints/Wan2.1-I2V-14B-480P

bash train_14B.sh
```

**Key Parameters:**

*   `CUDA_VISIBLE_DEVICES`:  GPU device selection.
*   `--pretrained_model_name_or_path`: Path to the pretrained Wan2.1-1.3B model.
*   `--pretrained_wav2vec_path`: Path to the pretrained Wav2Vec2.0 model.
*   `--output_dir`:  Directory to save trained checkpoints.
*   `--train_data_square_dir`, `--train_data_rec_dir`, `--train_data_vec_dir`:  Paths to the text files listing training data paths.
*   `--validation_reference_path`, `--validation_driven_audio_path`:  Paths to validation data.
*   `--video_sample_n_frames`: Number of frames processed per batch.
*   `--num_train_epochs`: Number of training epochs (set to infinite for continuous training, manually terminate when satisfied with performance).

### Lora Training/Finetuning Commands
```bash
# Training StableAvatar-1.3B on a mixed resolution setting (480x832 and 832x480) in a single machine
bash train_1B_rec_vec_lora.sh
# Training StableAvatar-1.3B on a mixed resolution setting (480x832 and 832x480) in multiple machines
bash train_1B_rec_vec_lora_64.sh
# Lora-Training StableAvatar-14B on a mixed resolution setting (480x832, 832x480, and 512x512) in multiple machines
bash train_14B_lora.sh
```
*   You can modify `--rank` and `--network_alpha` to control the quality of your lora training/finetuning.

### Model Finetuning
```bash
# Finetuning StableAvatar on a mixed resolution setting (480x832 and 832x480) in a single machine
bash train_1B_rec_vec.sh --transformer_path="path/StableAvatar/checkpoints/StableAvatar-1.3B/transformer3d-square.pt"
# Finetuning StableAvatar on a mixed resolution setting (480x832 and 832x480) in multiple machines
bash train_1B_rec_vec_64.sh --transformer_path="path/StableAvatar/checkpoints/StableAvatar-1.3B/transformer3d-square.pt"
```
*   You can modify `--rank` and `--network_alpha` to control the quality of your lora training/finetuning.

### Training Tips

*   Training StableAvatar requires a significant amount of VRAM.
*   Static backgrounds and clean audio are recommended.

## VRAM Requirement and Runtime

The basic model (480x832, fps=25) needs about 18GB VRAM and runs in about 3 minutes on a 4090 GPU for a 5-second video.
